HTML: HyperText Markup Language

HTML (HyperText Markup Language) is the most basic building block of the Web. It is the standard markup language for Web pages.
HTML (HyperText Markup Language) is the code that is used to structure a web page and its content.
HTML defines the meaning and structure of web content. Other technologies besides HTML are generally used to describe a web page's appearance/presentation (CSS) or functionality/behavior (JavaScript).

"Hypertext" refers to links that connect web pages to one another, either within a single website or between websites.

HTML uses "markup" to annotate text, images, and other content for display in a Web browser. HTML markup includes special "elements" such as <head>, <title>, <body>, <header>, <footer>, <article>, <section>, <p>, <div>, <span>, <img>, <aside>, <audio>, <canvas>, <datalist>, <details>, <embed>, <nav>, <output>, <progress>, <video>, <ul>, <ol>, <li> and many others which are used to enclose, or wrap, different parts of the content to make it appear a certain way, or act a certain way.

HTML Element

An HTML element is a component of an HTML document that tells a web browser how to structure and interpret a part of the HTML document. HTML elements can contain formatting instructions, semantic meaning, and content.
An HTML element is set off from other text in a document by "tags", which consist of the element name surrounded by "<" and ">". The name of an element inside a tag is case insensitive.

Anatomy of an HTML Element

The main parts of an element are as follows:

  1. The opening tag: This consists of the name of the element (in this case, p), wrapped in opening and closing angle brackets. This states where the element begins or starts to take effect — in this case where the paragraph begins.
  2. The closing tag: This states where the element ends — in this case where the paragraph ends. Failing to add a closing tag can lead to strange results.
  3. The content: This is the content of the element, which in this case, is just text.
Elements can also have attributes that look like the following:
Attributes contain extra information for the browser about the element to specify modifications to that HTML element. An attribute should always have the following:
  1. A space between it and the element name (or the previous attribute, if the element already has one or more attributes).
  2. The attribute name followed by an equal sign.
  3. The attribute value wrapped by opening and closing quotation marks.
Some HTML Elements can also be self-closing tags, such as <br>, <hr>, etc.

Anatomy of an HTML document

By convention, the HTML file of Home page of a website is named index.html
Let us consider the following index.html file

<!DOCTYPE html>
<html>
  <head>
  <meta charset="utf-8">
  <meta name="description" content="This is a web page for HTML basics notes">
  <meta name="author" content="Vibhu Manikpuri">
  <title> homepage </title>
  </head>
  <body>
  <h4> This is my home page </h4>
  </body>
</html>

  • <!DOCTYPE html> -- doctype is a required preamble and tells the browser the version of HTML being used in this file.
  • <html> </html> -- The <html> element wraps all the content on the entire page and is sometimes known as the root element. Everything wrapped inside this element is HTML code
  • <head> </head> -- The <head> is the part of the web page that holds the information about the web page and tells the browser how to handle the webpage. It acts as container for things like title, metadata, desription for web page when appearing in Search Engine, CSS to style the content of HTML page, and more
  • <title> </title> -- The <title> sets the title of the page, which is the title that appears in the browser tab the page is loaded in
  • <meta charset="utf-8"> -- The <meta> element contain metadata for our HTML page. <meta charset="utf-8"> sets the character set for HTML document to UTF-8. UTF-8 is the standard encoding that we should be using as it includes most characters from the vast majority of written languages. Other than character-set, this element also let's us add other meta data such as author, view port, etc
  • <body> -- The <body> element contains all the content that you want to show to web users when they visit the web page, which may include text, images, videos, games, or more

Fundamental HTML tags

Other than <html>, <head>, <title>, <meta> and <body> elements, following are some fundamental HTML elements or family of elements

  • Image
    <img src="images/path_to_image/directory/image.png" alt="this is an image">
    The img element embeds an image into our page in the position it appears. The src attribute contains the path to image file. The alt attribute hilds a text description of the image.
  • Headings
    Heading elements allow us to specify some parts of the HTML file as headings or sub-headings. HTML contains 6 heading levels <h1> - <h6>.
    <h1> Main Title </h1>
    <h3> Sub-Title </h3>
  • Paragraph
    This element is used for conataining paragraphs of text.
    <p> This is a single paragraph </p>
  • Lists
    HTML has special elements for lists. The most common list types are
    1. Unordered lists are for lists where the order of the items doesn't matter, such as a shopping list. These are wrapped in a <ul> element.
    2. Ordered lists are for lists where the order of the items does matter, such as ranks of teams. These are wrapped in a <ol> element.
    Each item inside the lists is put inside an <li> (list item) element.
  • Links Links are very important - they are what makes the web the web. Links are added to an HTML file using the <a> element.
    An <a> element, with it's href attribute creates a hyperlink to web pages, files, images, locations in the same page, or anything else a URL can address.
    Content within each <a> should indicate the link's destination.
    Steps to turn text within a paragraph into a link:
    1. Choose some text. Let's choose "this is my LinkedIn profile"
    2. Wrap the text in <a> element, as shown below:
      <a> this is my LinkedIn profile </a>
    3. Give the <a> element an href attribute, as shown below:
      <a href=""> this is my LinkedIn profile </a>
    4. Give the <a> element an href attribute that points to the URL or the image location, as shown below:
      <a href="https://www.linkedin.com/in/manikpurivibhu/"> this is my LinkedIn profile </a>
    this is my LinkedIn profile
  • The Content Division element
    The <div> HTML element is the generic container for flow content. It has no effect on the content or layout until styled in some way using CSS.
    As a "pure" container, the <div> element does not inherently represent anything. Instead, it's used to group content so it can be easily styled using the class or id attributes, marking a section of a document as being written in a different language (using the lang attribute), and so on.
    More on this (including it's code example) later when studying CSS.
  • Generic Section
    The <section> HTML element represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. Sections should always have a heading, with very few exceptions. (More on this later)
  • Break Line
    The <br> HTML element produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant.
  • Horizontal Rule
    The <hr> HTML element represents a thematic break between paragraph-level elements: for example, a change of scene in a story, or a shift of topic within a section.
  • Boldface element
    The <b> HTML element bolds a piece of text. However, it is recommended to use either CSS, or <strong> HTML element to draw reader's attention in a piece of text.
  • Strong Importance
    The <strong> HTML element indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
  • Idiomatic Text
    The <i> HTML element represents a range of text that is set off from the normal text for some reason, such as italicized text.
  • Emphasis
    The <em> HTML element marks text that has stress emphasis. This element can be nested, with each level of nesting indicating a greater degree of emphasis.
  • The Details disclosure element
    The <details> HTML element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state. A summary or label must be provided using the <summary> element.
    Details Something small enough to escape casual notice.
  • Table
    The <table> HTML element represents tabular data — that is, information presented in a two-dimensional table comprised of rows and columns of cells containing data.
    However, the <table> element can be used only via a set of other elements. These elements work in a heirarchical relationship.
    • <thead>: The Table Head Element defines a set of rows defining the head of the columns of the table.
    • <tbody>: The Table Body Element encapsulates a set of table rows (<tr> elements), indicating that they comprise the body of the table (<table>).
    • <tr>: The Table Row Element defines a row of cells in a table. The row's cells can then be established using a mix of <td> (data cell) and <th> (header cell) elements.
    • <td>: The Table Data Cell Element defines a cell of a table that contains data. It contains the actual data to be stored in the table cells.
    • <th>: The Table Header Element defines a cell as header of a group of table cells. The exact nature of this group is defined by the scope and headers attributes.
    • <caption>: The Table Caption element specifies the caption (or title) of a table.

  • Form
    The <form> HTML element represents a document section containing interactive controls for submitting information.
    The <form> element itself is unable to do anything, and requires other important HTML elements like <label>, <input>, etc to perform any tasks.
    <form action="/notes.html" method="post" >
      <label>Enter your name :</label>
      <input type="text" name="" value=""> <br>
      <label>Enter your email :</label>
      <input type="email" name="" value=""> <br>
      <input type="submit" name="">
    </form>


    • The action attribute defines the location (URL) where the form's collected data should be sent when it is submitted
    • The method attribute defines which HTTP method to send the data with (usually get or post)

    Following are some essential HTML elements that pair with <form> to make it so powerful

    • The <input> HTML element is used to create interactive controls for web-based forms in order to accept data from the user; a wide variety of types of input data and control widgets are available, depending on the device and user agent.
      We specify the type attribute to be text, radio ,button, checkbox, color, date, email, file, password, time, url, etc. to create various kinds of interactive controls for the form.

    • The <label> HTML element represents a caption for an item in a user interface. It adds a label to a form control such as text, email, password, textarea etc.

      The <label> element is associated with an <input> element such as text, email, etx of the form.

    To associate the <label> with an <input> element, you need to give the <input> an id attribute. The <label> then needs a for attribute whose value is the same as the input's id.

    <label for="first_name"> First Name :</label>
    <input type="text" id="first_name">

    The for attribute containes the first_name which matches the id of our input. So, this label is associated with this input element.
    Alternatlively, <input> can also be nested directly inside the <label>, in which case the for and id attributes are not needed because the association is implicit.
    <label> First Name :
      <input type="text" name="first_name">
    </label>
    The name attribute in <input> is used as a key when the form is submitted and it's data is sent as a key-value pair using POST request.

    • Button
      The <input> elements of type button are rendered as simple push buttons, which can be programmed to control custiom functionality anywheere on a webpage as required when assigned to an event handler function (typically the click event).
      <input type="button" value="click me">
    • Submit
      The <input> elements of type submit are rendered as buttons.When the click event occurs, the user agent attempts to submit the form to the server.
      <input type="submit" value="submit form">
    • Reset
      The <input> elements of type reset are rendered as buttons, with a default click event handler that resets all of the inputs in the form to their initial values.
      <input type="reset" value="clear">
